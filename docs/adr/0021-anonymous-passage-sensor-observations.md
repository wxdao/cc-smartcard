# Anonymous Passage Sensor Observations

Passage Sensors expose anonymous observations of every entity in their detection area except spectator players, so Lua programs can implement their own filtering rather than relying on a sensor-wide mode. This includes creatures, vehicles, projectiles, and dropped items. Observations include entity kind, a sensor-scoped temporary token, sensor-relative position and velocity, and collision size; they exclude player UUIDs, names, persistent entity identifiers, health, custom names, teams, equipment, inventories, and NBT. This preserves flexible passage logic without turning the sensor into a passive identity or entity-inspection device. A newly attached computer receives `passage_entity_entered` events for entities already present, then subsequent entry and exit changes, while `getEntities()` provides an authoritative snapshot for reconciliation. Changing the Detection Area or ending continuous detection resets the observation session: previous tokens become invalid, a reset is reported, and entities in the next active area receive new tokens instead of synthetic movement across configurations.
