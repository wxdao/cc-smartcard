# Both Cabinets Address One Access Gate

The two paired Gate Cabinets are equal access points to one logical Access Gate: a CC computer connected through either cabinet controls the same shared gate state. This avoids a hidden controller/passive distinction, although the two physical attachments may have different ComputerCraft peripheral names; shared gate identity and state events allow Lua programs to recognise that both attachments address the same gate. A computer attached through both cabinets receives each shared event once, deduplicated by its stable ComputerCraft computer ID, while distinct computers each receive the event. The Gate ID is an opaque value derived deterministically from the dimension and paired cabinet positions, avoiding persisted identity conflicts while remaining stable across reloads and replacement at the same location.

Both cabinets persist a copy of the shared actuator state because either cabinet may remain loaded when the other is across an unloaded chunk boundary. Each state mutation carries the persisted world game time plus an intra-tick sequence. When the pair is loaded together again, the newer version replaces the stale copy before the canonical cabinet advances the state machine. The fail-open state wins an otherwise equal-version conflict, preventing stale closed state from reversing a completed safety opening.
